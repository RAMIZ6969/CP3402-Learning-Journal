My Learning Experiment  

Hypothesis: I think that writing down my PHP code on a piece of paper before I type it into the computer will help me make fewer mistakes.

The Test: 1. I picked two similar coding tasks for my project.
2. For the first task, I just started typing directly into the code editor and fixed errors as they popped up.
3. For the second task, I sat away from the computer and wrote out the logic and the loops on a notepad first.
4. I tracked how many times I had to refresh the page to get the code working perfectly for each task.

The Results:  

Task 1 (Typing only): I had to refresh the page 8 times. Most of the errors were small things like missing semicolons or brackets because I was trying to code too fast.

Task 2 (Paper first): I only had to refresh the page 2 times. Because I had already thought about how the code should work, I made almost no mistakes when I finally typed it in.

Conclusion: My hypothesis was right. Planning on paper saves time because it stops me from guessing. It makes the actual coding part much smoother.
