## Content Insights
Through my research into Git configuration, I learned that Git requires a global user identity (email and name) before it allows any commits to be recorded. This ensures every save point has an author for accountability, which is vital for project management. I also discovered that when using PowerShell, folder paths containing spaces must be wrapped in quotation marks (e.g., cd "Week 1") because the terminal interprets spaces as the start of a new command parameter.

## Career/Employability/Learning Insights
This week taught me the importance of documentation and version control as professional skills rather than just academic tasks. By resolving the src refspec main does not match any error, I realized that many technical failures are caused by skipping foundational setup steps—specifically, the need to initialize (git init) and commit locally before attempting to push to a remote server. In a professional setting, being able to troubleshoot these terminal-level issues independently makes a developer much more employable and efficient.
I found that my learning process is most effective when I stop to research the underlying logic of an error rather than just trying to find an easier fix. For example, learning why PowerShell failed on the "Week 1" path has given me a lasting skill for navigating any command-line interface, which is a core competency for my future goal of becoming a Data Scientist.
