## Content Insights
A major insight this week was learning how Twenty Twenty-Five differs from traditional themes. It is a "Block Theme," meaning the entire site—including the header and footer—is built using the Block Editor.
I learned that the Site Editor (Appearance > Editor) is where you set the "global" styles, like the color palette and typography.
This is much more efficient than styling individual pages because it ensures that every new page I create for UniTasker automatically looks consistent.
Regarding functionality, I realized that using WPForms isn't just about creating a visual form; it's about data collection.
I learned that each form submission is saved as a record in the site's database. This is critical for a startup prototype because even if an email notification fails, the "business data" (the user's task request) is safely stored and accessible through the WordPress backend.

## Career/Employability/Learning Insights
This week shifted my perspective from being a "user" of technology to a "builder" of systems. I realized that a professional website isn't just a collection of pretty pages; it's an interface for a database.
As someone pursuing Data Science, I have learned that the front end of a system is actually the first step of Data Engineering. By setting up specific field types and required validation in WPForms, I am essentially cleaning and validating data before it ever reaches the database.
It highlighted for me that a Data Scientist needs to understand the entire pipeline—from how a user types in their name to how that string of text is stored in an SQL table
